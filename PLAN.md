## Финальный стек (утверждён)

| Слой | Технология | Зачем |
|---|---|---|
| Frontend | **Vite + React 18 + TypeScript** | На основе Lovable-прототипа Жанетты — не теряем готовый UI |
| UI | **Tailwind CSS + shadcn/ui** | Уже в Lovable-коде, профессиональный вид |
| State | **TanStack Query v5 + Zustand** | Server state и client state раздельно |
| Forms | **React Hook Form + Zod** | Лучшая практика 2026 года |
| Real-time | **Socket.IO Client v4** | Судейская панель и live-табло |
| Backend | **Fastify + TypeScript** | Быстрее Express, типобезопасный |
| ORM | **Prisma 5** | Тип-сейф, миграции, Studio для просмотра БД |
| БД | **PostgreSQL 16** | Локально через pgAdmin + Docker |
| Cache/PubSub | **Redis 7** | Через docker-compose |
| Auth | **JWT + bcrypt + cookie refresh** | Access 15 мин + Refresh 7 дней |
| Валидация | **Zod (общая для фронта и бэка)** | DRY-схемы |
| PDF | **PDFKit + Puppeteer** | Дипломы и протоколы |
| Email | **Nodemailer + Mailtrap (dev)** | Уведомления и инвайты |
| Деплой | **Vercel (frontend) + Render (backend + Postgres + Redis)** | Бесплатные tier-планы |
| Локальный запуск | **docker-compose** | Fallback для защиты без интернета |

## Управление рисками

1. **Bracket Engine (день 7) — самое сложное.** Если не успеваем — упрощаем: только Single Elimination без Repechage. Repechage добавим в день 14, если останется время.
2. **Real-time (день 8) — может буксовать.** Запасной план: вместо Socket.IO — polling раз в 2 секунды (визуально незаметно, на защите ок).
3. **Деплой бэка (день 16).** Если Render-tier тормозит — переключаемся на Railway или Fly.io. Локальный docker-compose всегда работает как fallback на ноутбуке.
4. **ЕНТ-подготовка.** Когда устаёшь от кода — переключайся на математику/информатику. Я могу прерваться в любой момент. Главное — двигаться каждый день.
