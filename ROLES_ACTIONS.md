# 🎭 Judo-Arena — Действия по ролям (для дизайна в Lovable)

Это твоя шпаргалка для Lovable. Здесь все экраны и кнопки, которые должны быть для каждой роли. Используй чтобы привести дашборды к **единому стилю** и убедиться что ничего не забыто.

> Общая идея: у всех 4 ролей дашборды должны выглядеть одинаково по структуре (sidebar или верхнее меню), отличаться только содержимым. Используй один `DashboardShell.tsx` для всех.

---

## 👤 1. PUBLIC (без авторизации)

Зрители, родители спортсменов, кто угодно. Доступ через главную страницу.

### Страницы
| Страница | URL | Что показать |
|---|---|---|
| Главная | `/` | Hero, ближайшие турниры, топ-спортсмены, клубы, как работает |
| Список турниров | `/tournaments` | Фильтры: статус (LIVE, регистрация, прошедшие), город, дата |
| Страница турнира | `/tournaments/:id` | Описание, категории, расписание, заявки (количество) |
| Сетки турнира | `/tournaments/:id/brackets/:categoryId` | Визуальная сетка с live-обновлением (Socket.IO) |
| Публичное табло (live) | `/live/:tournamentId` | Все идущие матчи на всех татами с real-time счётом |
| Рейтинг | `/rankings` | Leaderboard: топ спортсменов и клубов (фильтры по полу, весу, городу) |
| Клубы | `/clubs` | Список клубов |
| Страница клуба | `/clubs/:id` | Описание, спортсмены, история выступлений |
| Профиль спортсмена | `/athletes/:id` | Имя, клуб, рейтинг, история матчей |
| Логин/Регистрация | `/login` | Форма входа + переключатель на регистрацию |
| О проекте | `/about` | О Judo-Arena |

### Действия
- Просматривать всё (read-only)
- Войти / зарегистрироваться как **спортсмен** или **тренер**
- Переключать язык (RU / KZ / EN) — в правом верхнем углу

## 🥋 2. ATHLETE (Спортсмен)

После входа попадает в `/dashboard` или `/athlete`.

### Главный дашборд
**Структура:** Sidebar слева + контент справа

**Sidebar (меню):**
- 🏠 Главная (мой профиль)
- 🏆 Мои турниры
- ⚔️ Мои матчи
- 📊 Мой рейтинг
- ⚙️ Настройки
- 🚪 Выйти

### Страницы

#### `/athlete` — Главная (профиль)
**Карточки сверху (4 шт):**
- Сумма очков рейтинга
- Кол-во турниров
- Побед / поражений
- Место в общем рейтинге

**Блоки ниже:**
- Личная информация: имя, фамилия, дата рождения, пол, вес, пояс, клуб
- Кнопка **"Редактировать профиль"** → форма (PATCH `/api/athletes/:id`)
- Кнопка **"Сменить пароль"** (отдельный API в будущем)
- Кнопка **"Сменить язык"** (kk/ru/en)

#### `/athlete/tournaments` — Мои турниры
Список турниров где я участвую (через approved Application моего клуба).
**Колонки таблицы:** Название · Дата · Город · Категория · Статус (Up/Live/Done) · Мой результат (медаль/место)

Клик по строке → детали турнира.

#### `/athlete/matches` — Мои матчи
История всех моих матчей. **Колонки:** Дата · Турнир · Категория · Соперник · Результат (W/L) · Способ (Ippon/Wazaari/Decision) · Очки.

#### `/athlete/rating` — Мой рейтинг
- График роста очков по датам
- Таблица: турнир · категория · место · очки
- Сравнение с топом

#### `/athlete/settings` — Настройки
- Язык интерфейса
- Email (только просмотр)
- Кнопка **"Выйти со всех устройств"** (POST `/api/auth/logout-all`)

### Что НЕ может ATHLETE
- Создавать клубы, турниры, заявки
- Управлять другими пользователями
- Видеть админ-функции

## 🥋 3. COACH (Тренер)

После входа попадает в `/coach`.

### Главный дашборд
**Sidebar:**
- 🏠 Мой клуб
- 👥 Спортсмены
- 📋 Заявки на турниры
- 🏆 Турниры (просмотр)
- 📊 Рейтинг клуба
- ⚙️ Настройки
- 🚪 Выйти

### Страницы

#### `/coach` — Мой клуб
**Если клуба нет:**
- Большая кнопка **"Создать клуб"** → форма:
  - Название (3 поля для kk/ru/en)
  - Краткое название (slug)
  - Город · Страна
  - Логотип (URL)
  - Описание (3 поля)
  - **[Создать]** → POST `/api/clubs`

**Если клуб есть:**
- Шапка клуба: логотип, название, город, кол-во спортсменов
- Кнопка **"Редактировать клуб"** (PATCH `/api/clubs/:id`)
- Блок "Возрастные группы":
  - Список групп (имя, диапазон возраста)
  - **[+ Добавить группу]** → форма (POST `/api/clubs/:id/groups`)
  - На каждой группе: иконки ✏️ редактировать, 🗑️ удалить

#### `/coach/athletes` — Мои спортсмены
- **[+ Добавить спортсмена]** → форма прокси-регистрации:
  - Email · Начальный пароль
  - Имя · Фамилия (и опционально на латинице)
  - Дата рождения · Пол
  - Вес (кг) · Пояс
  - Язык по умолчанию
  - **[Добавить]** → POST `/api/clubs/:id/athletes`
- Таблица спортсменов: имя · пол · возраст · вес · пояс · действия
- На каждой строке: ✏️ редактировать профиль (PATCH `/api/athletes/:id`) · ❌ отвязать от клуба (DELETE `/api/athletes/:id/club`)

#### `/coach/applications` — Заявки на турниры
**Список заявок** моего клуба со статусами: DRAFT, SUBMITTED, APPROVED, REJECTED, WITHDRAWN.

Клик по заявке → детали:

**Страница заявки `/coach/applications/:id`:**
- Шапка: турнир, статус, дата отправки
- Если **DRAFT**:
  - Кнопка **[+ Добавить спортсмена]** → выбрать спортсмена и категорию → POST `/api/applications/:id/entries`
  - Кнопка **[🚀 Отправить заявку]** → POST `/api/applications/:id/submit`
  - На каждой записи: ❌ убрать (DELETE `/api/applications/:id/entries/:entryId`)
- Если **SUBMITTED**:
  - Кнопка **[↩ Отозвать]** → POST `/api/applications/:id/withdraw`
- Если **APPROVED/REJECTED**:
  - Только просмотр + комментарий админа

#### `/coach/tournaments` — Турниры (просмотр)
Список всех турниров. Для каждого — кнопка **[📝 Подать заявку]** (если статус REGISTRATION_OPEN).
- Нажатие → POST `/api/tournaments/:id/applications` (создаёт DRAFT)
- Затем переход на `/coach/applications/:id` для заполнения

#### `/coach/rating` — Рейтинг клуба
Топ моих спортсменов: имя · турниры · сумма очков · лучшие места.

### Что НЕ может COACH
- Управлять чужими клубами
- Одобрять заявки (это ADMIN)
- Создавать турниры
- Override результатов

## ⚖️ 4. JUDGE (Судья) — **БЕЗ АККАУНТА**

Судья получает от админа ссылку `/judge/:token` (по WhatsApp, Telegram, QR-коду). Открывает её на телефоне — попадает на **мобильный интерфейс** судейской панели.

> **Мобильный first.** Большие кнопки. Минимум текста. Адаптив от 320 px.

### Страница `/judge/:token`

**Шапка:**
- Название турнира
- Категория · Татами #N
- Имя судьи (если введено админом)

**Информация о матче:**
- 🔴 RED: имя + клуб
- 🔵 BLUE: имя + клуб

**Таймер матча:** большие цифры (например `04:00`) с обратным отсчётом.

**Табло счёта (большое):**
```
┌──────────────┐  ┌──────────────┐
│  🔴 RED      │  │  🔵 BLUE     │
│              │  │              │
│   IPPON  0   │  │   IPPON  0   │
│   WAZ    1   │  │   WAZ    0   │
│   SHIDO  0   │  │   SHIDO  1   │
└──────────────┘  └──────────────┘
```

**Кнопки управления (по 2 для RED и BLUE):**

Если матч не начат:
- **[🎌 ХАДЖИМЕ]** → POST `/api/matches/:id/start`

Если матч идёт:
- **[⏸ МАТЕ]** → POST `/api/matches/:id/pause`
- **[🥇 GOLDEN SCORE]** → POST `/api/matches/:id/golden-score` (когда основное время вышло и ничья)

**Кнопки начисления очков (RED):**
| Кнопка | Что делает |
|---|---|
| **IPPON** | POST `/score {type:"IPPON", side:"RED"}` → авто-победа |
| **WAZA-ARI** | POST `/score {type:"WAZA_ARI", side:"RED"}` |
| **SHIDO** | POST `/score {type:"SHIDO", side:"RED"}` (3-й = дисквалификация) |

Те же 3 кнопки для BLUE.

**Osaekomi (удержание):**
- **[🤼 OSAEKOMI RED]** → POST `/osaekomi {side:"RED"}` → стартует таймер 0:00 → 0:05 (yuko) → 0:10 (wazari) → 0:20 (ippon)
- Когда удержание активно — кнопка меняется на **[STOP TOKETA]** → POST `/toketa`
- На экране UI-таймер удержания крупным шрифтом красным цветом

**Кнопка ручного завершения:**
- **[🏁 SORE-MADE]** → POST `/finish {winnerSide: "RED"}` (если нужно вручную)

**После завершения матча:**
- Большой баннер **🏆 ПОБЕДИТЕЛЬ: <имя>**
- Кнопка **[✓ Завершить]** — закрыть панель
- Токен становится бесполезен (один матч = одна сессия)

### Что НЕ может JUDGE
- Видеть других пользователей
- Видеть админ-функции
- Использовать токен для другого матча

## 👑 5. ADMIN (Администратор)

Главный пользователь системы. После входа → `/admin`.

### Главный дашборд
**Sidebar (крупное меню):**
- 📊 Обзор (метрики)
- 🏆 Турниры
- 🥋 Клубы
- 👥 Пользователи
- ⚖️ Татами и матчи (live)
- 📋 Журнал аудита
- ⚙️ Настройки системы
- 🚪 Выйти

### Страницы

#### `/admin` — Обзор (Dashboard)
**Карточки сверху:**
- 🏆 Активных турниров (статусы: REGISTRATION_OPEN, IN_PROGRESS)
- 👥 Всего спортсменов
- 🥋 Всего клубов
- 📊 Всего матчей за месяц

**Блоки:**
- "Идущие турниры" — список с быстрым переходом на админ-страницу
- "Последние действия" — лента из AuditLog (5 последних)
- "Незакрытые заявки" — список SUBMITTED заявок ждущих approve

#### `/admin/tournaments` — Турниры

**Список** со всеми статусами + фильтры.

**[+ Создать турнир]** → форма:
- Название (3 языка) · Описание (3 языка)
- Город · Место проведения
- Дата начала · Дата окончания
- Число татами
- Основной язык
- Постер (URL)
- POST `/api/tournaments`

**Страница турнира `/admin/tournaments/:id`:**

Вкладки:
1. **Обзор** — описание, даты, статус
2. **Категории** — список + **[+ Добавить категорию]**
3. **Заявки** — все заявки клубов (SUBMITTED ждут одобрения)
4. **Сетки** — список сеток по категориям
5. **Татами** — live-управление очередью
6. **Аудит** — все действия по этому турниру

**Кнопки управления (зависят от статуса):**
- DRAFT → **[🔓 Открыть регистрацию]**
- REGISTRATION_OPEN → **[🔒 Закрыть регистрацию]**
- REGISTRATION_CLOSED → **[▶️ Начать турнир]** (status → IN_PROGRESS)
- IN_PROGRESS → **[🏁 Финализировать]** (POST `/api/admin/tournaments/:id/finalize`)
- Любой → **[❌ Отменить]**

##### Категории — форма создания
- Название (3 языка, опц.)
- Пол · Возраст от/до · Вес от/до
- Длительность матча (сек, default 240)
- Длительность Golden Score (default 0 = без лимита)
- Формат: **Single Elimination + Repechage** / **Round-Robin** / **Mixed**
- ✓ Разрешить Юко (старое правило)
- POST `/api/tournaments/:id/categories`

##### Заявки клубов
Таблица: клуб · кол-во спортсменов · статус · дата отправки.
SUBMITTED заявки имеют кнопки:
- **[✅ Одобрить]** → POST `/api/applications/:id/approve` (можно добавить заметку рецензента)
- **[❌ Отклонить]** → POST `/api/applications/:id/reject`

##### Сетки
По каждой категории:
- Если сетка не создана: **[⚙️ Сгенерировать сетку]** → POST `/api/.../bracket`
- Если создана:
  - Визуальная сетка (компонент Bracket.tsx из Lovable)
  - **[📄 Скачать PDF сетки]** → `/api/admin/bracket?bracketId=...`
  - **[🗑️ Удалить сетку]** (только если матчи не начались)

##### Татами и матчи (live)
Для каждого татами (1, 2, 3...):
- Текущий матч (если IN_PROGRESS) — большой счёт, real-time через Socket.IO
- Очередь следующих матчей
- Drag-and-drop для перестановки порядка
- На каждом матче: **[➕ Создать сессию судьи]** → выдаёт URL для копирования

##### Кнопка финального протокола
После COMPLETED: **[📄 Скачать PDF протокола]** → `/api/admin/protocol?tournamentId=...`

#### `/admin/clubs` — Клубы
Полный CRUD. Создать клуб для тренера (если тренер не зарегистрировался), удалить клуб.

#### `/admin/users` — Пользователи
Таблица всех пользователей: email · имя · роль · клуб · активен.
- Поиск по email/имени
- Фильтр по роли
- Деактивация аккаунта (PATCH с `isActive: false`)

#### `/admin/matches` — Live матчи + Override

**Текущие живые матчи** (status=IN_PROGRESS) — карточки с real-time счётом.

**Завершённые матчи** — фильтр по дате/турниру. На каждой записи:
- **[✏️ Override]** → форма:
  - Выбрать нового победителя (RED/BLUE)
  - Обязательное поле "Причина" (минимум 3 символа)
  - **[Применить + Rollback]** → POST `/api/admin/matches/:id/override`
- Предупреждение: "Это откатит все downstream-матчи где играл прежний победитель"
- После override: список откаченных матчей

#### `/admin/audit-logs` — Журнал аудита
Таблица: дата · кто · действие · сущность · изменение.
Фильтры: targetEntity (Match/Tournament/Application), action (override/rollback/finalize), actor.
Раскрытие строки → JSON before/after.

#### `/admin/settings` — Настройки системы
- Настройка шкалы очков (SystemConfig.ratingPoints):
  - 1 место = 100 (редактируемо)
  - 2 = 80
  - 3 = 50
  - и т.д.
- Бонус за Ippon (опц.)
- Включить/выключить Юко глобально
- Сохранить → PATCH `/api/system-config/ratingPoints` (этот эндпоинт можно добавить позже)

## 🎨 Общие элементы UI (для всех ролей)

### Шапка (Header)
- Лого Judo-Arena (слева)
- Меню (центр или скрыто для зарегистрированных)
- Переключатель языка 🇰🇿 / 🇷🇺 / 🇬🇧 (правый верх)
- Уведомления 🔔 (для зарегистрированных, бейдж с числом непрочитанных)
- Профиль с выпадающим меню (моя информация / настройки / выйти)

### Подвал (Footer)
- Лого · Краткое описание
- Ссылки: О проекте · Турниры · Рейтинг · Клубы
- Контакты · Соцсети

### Цветовая палитра (как в твоём Lovable)
- Основа: глубокий navy `#0B1426`
- Акцент: gold `#D4AF37` (для кнопок, ссылок, медалей)
- Текст: белый/серый на тёмном фоне
- RED side: красный (для красного спортсмена)
- BLUE side: синий (для синего спортсмена)
- Success: emerald
- Danger: rose

### Адаптив
- **Главная и публичные страницы:** desktop-first, но красиво от 768 px
- **Все дашборды (Athlete/Coach/Admin):** desktop-first
- **Судейская панель `/judge/:token`:** **mobile-first** (320 px+). На десктопе можно показать в центре экрана как карточку.

### Состояния пустых данных
Везде где может быть пустой список — красивый empty state с иконкой и кнопкой действия:
- "У вас пока нет турниров → [Найти турнир]"
- "Спортсменов в клубе нет → [+ Добавить первого]"
- "Заявок не подано → [Подать заявку на турнир]"

### Ошибки
Toast-уведомления (snackbar) внизу:
- ✅ Успех (зелёный): "Заявка отправлена"
- ❌ Ошибка (красный): "Спортсмен не подходит по весу"
- ℹ️ Инфо (синий): "Сетка сгенерирована, можно скачать PDF"

### Loading states
- Скелетоны при загрузке таблиц/карточек
- Спиннер на кнопках во время операции
- Disabled state на кнопках во время запроса

## 🌐 Многоязычность (i18n)

**Все** тексты UI хранятся в `client/src/shared/locales/{ru,kk,en}/*.json`.

Используется `react-i18next` (добавим завтра при подключении к API).

Файлы переводов:
- `common.json` — кнопки, общие тексты (Сохранить, Отмена, Поиск, и т.д.)
- `auth.json` — экраны логина/регистрации
- `tournament.json` — турниры, категории
- `judge.json` — судейская панель
- `admin.json` — админка
- `athlete.json` — кабинет спортсмена
- `coach.json` — кабинет тренера

Переключатель в шапке. Выбор сохраняется в localStorage **и** в профиле пользователя (PATCH `/api/auth/me/locale`).

- [ ] Судейская панель адаптивна для мобильного (большие кнопки)

- [ ] Все таблицы имеют пустое состояние и пагинацию
- [ ] Toast-уведомления для всех действий

## 🎯 Завтрашний план

1. Утром ты в Lovable приводишь дашборды к единому виду по этому чеклисту
2. Экспортируешь обновлённый код на GitHub
3. Я подтягиваю изменения в `web/` (`git pull` в Lovable-проекте)
4. **Подключаем фронт к работающему backend:**
   - axios-клиент с auto-refresh токенов
   - React-роутер (или TanStack Router, который уже в Lovable)
   - TanStack Query для server state
   - Zustand для client state (текущий пользователь, тема, язык)
   - Socket.IO Client для real-time
   - i18next с переводами
   - Подключаем формы к API эндпоинтам
5. Тестируем end-to-end сценарий с реальным UI
